Sass and Drupal: A Brief Introduction

25th October, 2013

Produce cleaner, more manageable CSS with SASS and add it to your existing Drupal workflow.

For those familiar with Sass, you’ll know just how much easier, and more importantly, how much more fun it can make working on CSS. It removes so many of the typical frustrations that come along with working on CSS, and the biggest perk is that if you already know CSS, even at a basic level, picking up Sass is incredibly easy.

How can SASS help you?

The key features it provides are variables, nesting, mixins and inheritance. All of those mechanisms together allow you to write clean, easy to read and most important of all, reusable code. Sass files use the file extensions .scss or .sass.

A quick n’ dirty example of what a snippet might look like in Sass:

$my_width: 400px;
$my_color: #FF3399;

#my-div {
  .inner {
    width: $my_width * 2;

    p {
      color: lighten($my_color, 10%);
    }
  }
}

So just what’s going on here? Looks very similar to CSS, right? With the introduction of the reusable variables $my_color and $my_width along with some nesting, this allows us to produce much more elegant code. Once compiled, it would look like this:

#my-div .inner {
  width: 800px;
}

#my-div .inner p {
  color: #ff66b3;
}

You’ll notice that in addition to using variables to store the width and color values, you can implement logic and functions to perform other coolness. In the above example, the width is multiplied and the lighten function is being used to change the color!

Here is another example, using a mixin to create a simple, reusable snippet of code:

@mixin no-list {
    padding: 0;
    margin: 0;
    list-style: none;
}

ul.some-list {
    @include no-list;
}

ul.some-other-list {
    @include no-list;
}

As you can see, the mixin “no-list” contains a snippet that we may want to use in multiple places throughout the document. This would compile to:

ul.some-list {
  padding: 0;
  margin: 0;
  list-style: none;
}

ul.some-other-list {
  padding: 0;
  margin: 0;
  list-style: none;
}

Great stuff, right? Well, this is just a flavour of what you can do with Sass. Integrating Sass into your workflow allows you to achieve more while reducing the repetitive nature of traditional CSS.

So, where does Drupal come in?

Out of the box, Drupal doesn’t come with any kind of Sass implementation, so it’s up to you to decide which approach to take and work that into your theme. The official implementation is coded in Ruby, and you need to configure it to “watch” your Sass files so they can be compiled. Other implementations exist in PHP, and with a combination of a few contrib modules you can get Sass up and running and implemented in Drupal with ease. These are what we’ll be looking at today.

To set up Drupal with PHP-based Sass processing, you’ll need the following modules: Prepro (http://www.drupal.org/project/prepro), Sassy (http://drupal.org/project/sassy) and Libraries (http://www.drupal.org/project/libraries). You’ll also need to grab the latest build of PHPSass (https://github.com/richthegeek/phpsass).

To get all this working, just install the modules as per usual and drop the PHPSass library into sites/all/libraries and Prepro/Sassy will pick up the library and allow you to use it.

The prepro configure page (admin/config/media/prepro)

Once everything is installed and enabled, you’ll be able to configure Prepro to use Sassy and PHPSass to compile all your .scss and .sass files.

Next up, depending on the theme you are using, you’ll have to go into the info file and add a few lines so it’ll pick up and use whatever .scss files you want to be compiled. In Omega it’ll look something like this:

css[layout.scss][name] = layout.scss
css[layout.scss][description] = My SASS stylesheet!
css[layout.scss][options][weight] = 1

Generally speaking, as long as the .scss file(s) are being picked up by the theme, Prepro will step in and compile them when required. You’ll need layout.scss to exist within the right folders in your theme, but once this has all been set up you’ll be able to start using Sass in your Drupal theme!

SASS-Enabled Drupal Themes

There are a handful of Sass-ready themes available for your consumption if you’d prefer to use one of those. Zen (http://drupal.org/project/zen) and Boilerplate (http://drupal.org/project/boilerplate) both ship with Sass support, and Omega will add support in the near future.

There are also themes like Sasson (http://drupal.org/project/sasson) that includes a Sass compiler.

Conclusion

Hopefully this short article has helped you understand the very basics of what Sass can do for you and how easy it is to work into your next (or existing!) project.

In the future I’ll be doing a more detailed series on how we use Sass to supercharge our Drupal theming, and keep an eye out for a talk at Drupal Yorkshire! (http://drupalyorkshire.org.uk/)

Happy Sassing!
