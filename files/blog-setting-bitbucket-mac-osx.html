Setting up BitBucket and Git for Drupal Development on Mac OSX

Preface

I’ve used iMacs and MacBooks at various points in my career (the last time was during a period when my work was more design-focused) and ultimately at the time, I didn’t feel that it was a good fit for me. Having owned a MacBook Pro at home for a while now and loving the revamped experience, I recently decided to give it another go at work and have since found it to be much more rewarding.

Installing Git and setting up BitBucket

We use BitBucket as our source code host of choice at Curve and the following Guide will walk you through setting up BitBucket on Mac OS X (note the Mac I’m using has Mac OS X 10.8.4 installed). I’ll be using the Terminal within OS X in this guide, but if you’re new to using the Terminal don’t worry, I’ll explain it all as we go.

Before we begin, we first need to install Git, which can be downloaded from the following URL: http://git-scm.com/downloads.

Once Git is installed, open up a new instance of Terminal in OS X. We can access Terminal in two ways -  either the Applications folder in the Finder, or by typing Cmd+Space followed by entering the word Terminal in the window that pops up.

Note: SSH is required in this guide and luckily for recent MAC OS X users it’s installed by default - as such installing SSH is beyond the scope of what we’ll be discussing here today since I’m focussing on setting up BitBucket for Mac.

First up, we want to add our username and email to Git so that it can properly label any commits we make - we can do this by typing the following commands one after the other:

  1. git config --global user.name “Username”
  2. git config --global user.email “user@email.com”

Once our username and email are configured, we want to navigate to our ssh folder and we can do so by typing the following command ‘cd ~/.ssh’. If for some reason the folder doesn’t exist, you can create it at this point by typing ‘mkdir ~/.ssh’.

From here, we want to set up our SSH identity for BitBucket. At the prompt type ‘ssh-keygen’ - you’ll then be asked where you want the file to be saved - when prompted type ‘bbk_rsa’.

Next, you’ll be asked to enter a passphrase - I chose to leave this blank by simply hitting enter - if you want, type a password of your choosing at this point, or leave it blank as I did. You’ll be asked to confirm the passphrase and you’ll want to enter the same passphrase again and hit enter - if you also chose to leave the first blank, this is just a case of hitting enter again.

To ensure the files have been saved in the correct location, if you type the following command ‘ls ~/.ssh’ you should be presented with a directory listing of your ‘.ssh’ directory that should now contain two new files - namely ‘bbk_rsa’ and ‘bbk_rsa.pub’.

We now want to load our new identity into the SSH agent management program. To do this, type the following command ‘ssh-add ~/.ssh/bbk_rsa’. If the load is successful, you should see the following ‘Identity added...’ message.

To see a list of all the keys the agent is currently managing enter the command ‘ssh-add -l’ and you’ll be presented with a listing of all keys.

...now we’re ready to add our key to BitBucket

Load up http://bitbucket.org in your browser, login and navigate to ‘manage account’ > ‘ssh keys’ (at the time of writing this, the URL should be something similar to: https://bitbucket.org/account/user/username/ssh-keys/). You should then be presented with the following:

We want to copy the contents of our public key file into BitBucket. To do this, we need to return to the Terminal and type the following ‘pbcopy < ~/.ssh/bbk_rsa.pub’ and the contents of this file will then be copied to the clipboard.

Next, return to BitBucket, click the ‘Add key’ and press Ctrl+V to enter your key in the dialogue window that pops up. Give the key a memorable label and finally click ‘Add key’.

That’s essentially it. You’re now ready to commit and pull data to your hearts content. One final point that’s worth mentioning is that any repositories that exist within your BitBucket account need to be changed from HTTPS to SSH, and this can be done on the main repository page in the right hand column (highlighted below).
